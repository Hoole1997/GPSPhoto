## Design Context

### Users
This is a simple Street View lookup tool for people who need to quickly verify whether a coordinate has usable Google Street View imagery before opening an interactive panorama. Likely users include product builders, property researchers, operations teams, map data reviewers, and anyone comparing real-world locations by coordinates.

### Brand Personality
Practical, precise, calm. The product should feel like a focused inspection desk rather than a marketing page.

### Aesthetic Direction
Light, map-forward, utility-oriented interface with crisp controls, strong status feedback, and a large first-screen viewer. Avoid decorative hero layouts, generic SaaS cards, and dark neon map aesthetics.

### Design Principles
- Put the Street View result first; controls should support inspection without competing with the viewer.
- Make validation explicit before loading an embed, especially for unavailable or denied API responses.
- Keep API key handling transparent and local to the browser.
- Support fast repeated checks through history, favorites, and shareable location state.
- Use restrained color to communicate status, not decoration.
